# QuantTrade 开发说明书 v7.0

> 基于原始需求文档整理的开发版说明书。
> 原文件 `README.md` 保留不变，后续开发默认以本文件为准。

## 1. 项目目标

构建一个运行在 Mac mini 上的个人自动化交易系统（ATS），用于：

- 基于明确、可解释的量化规则执行交易；
- 将策略、风控、回测、执行、监控拆分为独立模块；
- 在保证风险边界的前提下，实现从回测到实盘的参数化部署；
- 提供专业、冷静、高信息密度的操作界面。

### 1.1 核心原则

- 去情绪化：系统不做主观预测，只对预设条件做确定性响应。
- 可解释性：禁用黑盒策略，所有逻辑必须参数化且可审计。
- 风控优先：账户级风控高于任何交易信号。
- 可回测、可部署：同一套策略定义应尽量复用于回测与实盘。

## 2. 产品范围

### 2.1 MVP 范围

第一阶段建议只交付以下闭环能力：

- 行情采集与本地存储；
- 单策略回测；
- 单策略模拟执行；
- 账户级与标的级风控；
- 参数配置管理；
- 基础监控面板与审计日志；
- Schwab 接口接入与认证续期；
- 消息推送（Telegram 或微信二选一即可）。

### 2.2 暂不纳入 MVP

- 多策略组合优化器；
- 行业级风险热力图高级分析；
- 多账户管理；
- 自动机器学习调参；
- 高频级别撮合模拟。

## 3. 系统模块拆分

建议按以下模块开发：

| 模块 | 职责 | 输出 |
| :--- | :--- | :--- |
| `market-data` | 拉取、清洗、缓存行情数据 | 标准化 OHLCV / 指标输入 |
| `strategy-engine` | 计算信号、持仓状态、出入场规则 | 买卖信号、止损位、理由 |
| `risk-engine` | 账户级和标的级风控校验 | 是否允许下单、风控告警 |
| `execution-engine` | 模拟盘/实盘下单、撤单、对账 | 订单结果、持仓同步 |
| `backtest-engine` | 历史回放、绩效统计、压力测试 | 回测报告、指标曲线 |
| `config-center` | 参数集管理、版本化 | 策略参数与运行配置 |
| `audit-log` | 记录每一步决策与异常 | 可审计日志流 |
| `dashboard` | 展示净值、风险、日志、策略状态 | UI 面板 |
| `notifier` | 推送交易动作与风险事件 | 手机告警消息 |

## 4. 策略系统设计

### 4.1 设计要求

- 所有策略必须暴露关键参数；
- 所有信号必须可回放、可解释；
- 所有策略必须实现统一接口，便于回测与实盘共用。

### 4.2 建议统一接口

每个策略至少实现：

- `generate_signal(market_state, position_state, params)`
- `calc_stop_loss(market_state, position_state, params)`
- `calc_position_size(account_state, market_state, params)`
- `explain_decision(context)`

### 4.3 默认策略：ATR 动态趋势追踪（ATR-DTF）

适用环境：

- 牛市；
- 强动量个股；
- 高 Beta 标的。

关键参数：

- `entry_donchian_n`：入场窗口，例如 20；
- `exit_donchian_m`：离场窗口，例如 10；
- `atr_smooth_period`：ATR 平滑周期；
- `risk_coefficient_k`：ATR 止损乘数；
- `adx_trend_filter`：趋势强度门槛，例如 25。

核心逻辑：

1. 当价格突破 `entry_donchian_n` 高点，且 ADX 高于门槛时，允许入场。
2. 入场后按 `k * ATR` 计算初始或动态止损位。
3. 当价格跌破 `exit_donchian_m` 低点，或触发止损时离场。
4. 所有决策必须输出解释文本，写入审计日志。

### 4.4 扩展策略库

| 策略 | 适用环境 | 关键参数 | 说明 |
| :--- | :--- | :--- | :--- |
| `MeanRev` | 震荡、低波、蓝筹 | `z_score_entry`, `bollinger_std`, `rsi_ob_os` | 捕捉偏离均值后的回归机会 |
| `ORB` | 财报季、高波动股 | `range_minutes`, `volume_surge_factor` | 用开盘区间突破结合放量确认 |
| `Squeeze` | 突破前夕 | `kc_bb_ratio`, `squeeze_release_n` | 监控压缩后释放 |
| `TimeStop` | 通用 | `max_hold_bars_unprofitable` | N 个周期不盈利则退出 |

## 5. 风险控制要求

风控优先级必须高于策略信号，任何风控命中都应阻止下单。

### 5.1 账户级风控

- `max_daily_drawdown`：默认 3%，触发后停止当日新开仓；
- `global_max_exposure`：例如 80%，限制总仓位上限；
- `max_open_positions`：限制同时持仓数量；
- `daily_loss_limit_amount`：支持绝对金额维度的日亏损上限。

### 5.2 标的级风控

- `slippage_tolerance`：最大允许滑点；
- `liquidity_filter`：最低日均成交额或成交量；
- `max_symbol_weight`：单一标的最大资金占比；
- `halt_or_gap_guard`：停牌/极端跳空保护。

### 5.3 仓位计算

```text
quantity = min(
  equity * risk_pct / (k * atr),
  equity * max_weight / price
)
```

要求：

- 任何订单发出前必须再次计算仓位；
- 仓位结果必须经过最小交易单位和可用现金校验；
- 若计算结果小于最小可交易手数，则放弃下单并记录原因。

## 6. 回测系统要求

### 6.1 必备能力

- 历史 K 线回放；
- 手续费与滑点模拟；
- 持仓与现金曲线计算；
- 回测参数批量扫描；
- 结果可导出。

### 6.2 必备指标

- 总收益率；
- 年化收益率；
- 最大回撤；
- Sharpe Ratio；
- Sortino Ratio；
- Win Rate；
- Profit Factor；
- 平均持仓时间；
- 最差连续亏损次数；
- 最大水下时间。

### 6.3 压力测试

支持对核心参数做网格扫描，例如：

- `risk_coefficient_k` 从 1.5 到 3.0；
- `entry_donchian_n` 从 10 到 55；
- `adx_trend_filter` 从 15 到 35。

输出要求：

- 收益稳定性曲线；
- 参数热力图；
- 最优参数与鲁棒参数对比。

## 7. 实盘执行要求

### 7.1 券商接入

当前目标券商为 Schwab，至少支持：

- 账户信息读取；
- 持仓读取；
- 下单；
- 撤单；
- 历史订单查询；
- OAuth2 自动续期。

### 7.2 执行安全约束

- 默认先做模拟盘通路；
- 实盘模式必须显式切换；
- 每次启动后应先与券商仓位做一次对账；
- 若本地状态与券商状态不一致，应进入保护模式并报警；
- 网络恢复后必须执行一次状态重同步。

---

## 8. 数据与存储

### 8.1 数据源要求

- 分钟级行情为主；
- 支持日线聚合；
- 支持指标预计算缓存；
- 所有时间戳统一时区处理。

### 8.2 存储建议

- 行情数据库：`DuckDB`
- 配置与参数集：`YAML` 或 `JSON`
- 审计日志：结构化日志文件或轻量数据库表
- 回测结果：本地文件 + 数据表索引

### 8.3 数据表建议

至少包含：

- `bars`
- `indicators`
- `orders`
- `fills`
- `positions`
- `account_snapshots`
- `backtest_runs`
- `parameter_sets`
- `audit_events`

---

## 9. UI / UX 要求

目标风格：专业、克制、信息密度高，偏 QuantConnect 风格。

### 9.1 视觉规范

- 主背景：`#0F111A`
- 正常运行色：`#4589FF`
- 风险预警色：`#FFB020`
- 错误/熔断色：建议增加 `#D64545`

### 9.2 核心界面

1. 参数控制台
2. 审计日志台
3. 账户净值曲线
4. 当前持仓与风险概览
5. 策略表现看板

### 9.3 参数控制台要求

- 每个参数支持滑块和精确输入；
- 参数变更需实时预览影响；
- 参数集支持保存、复制、命名、回滚。

### 9.4 审计日志要求

日志必须包含：

- 时间；
- 标的；
- 策略名；
- 信号状态；
- 风控校验结果；
- 下单结果；
- 解释文本。

示例：

```text
[2026-03-27 09:31:05] AAPL ATR-DTF signal=LONG_ENTRY check=PASS reason="price > donchian_20_high and adx > 25"
```

## 10. 非功能性要求

### 10.1 稳定性

- 支持 Mac mini 长时间运行；
- 模块异常不应导致全系统崩溃；
- 关键任务需要自动重试；
- 启动后自动完成健康检查。

### 10.2 可观测性

- 关键操作必须写日志；
- 支持错误分级；
- 支持基础健康状态检测；
- 推送必须区分信息、警告、严重错误。

### 10.3 可维护性

- 模块边界清晰；
- 核心逻辑可单测；
- 策略新增不应破坏已有引擎；
- 配置变更应尽量无代码化。

## 12. 验收标准

以下条件满足时，可认为 MVP 可进入下一阶段：

- 能导入并存储目标标的分钟线；
- 能完成 ATR-DTF 的历史回测；
- 能输出完整回测报告；
- 能基于参数集生成交易信号；
- 能执行账户级与标的级风控拦截；
- 能在模拟环境完成下单与持仓更新；
- 能展示基础净值曲线、持仓和日志；
- 能通过 Schwab 完成认证与基础账户同步；
- 能通过手机收到关键风险和交易通知。
